## 1. 事前準備

作業を開始する前に、以下の準備が完了していることを確認してください。

- **LINE Business ID:** LINE Business IDが作成済みであること。
- **管理コンソール:** 以下の2つの管理コンソールの役割を理解しておくこと。
  - **[LINE Official Account Manager](https://manager.line.biz/):** LINE公式アカウントの管理やメッセージ配信設定を行います。
  - **[Line Developer Console](https://developers.line.biz/console/):** プロバイダーやチャンネルの作成、Messaging APIの設定など、開発者向けの設定を行います。

## 2. プロバイダーの作成

> **Note:**
> 既存のプロバイダーを利用する場合や、すでに作成済の場合は本手順をスキップしてください。
> 複数のチャンネルを単一のプロバイダーに紐づけることが可能です。

1. **[Line Developer Console](https://developers.line.biz/console/)** にアクセスします。
2. 「プロバイダー作成」をクリックします。
3. プロバイダー名を入力し、「作成」をクリックします。

## 3. チャンネルの作成

1. **[LINE Official Account Manager](https://manager.line.biz/)** にアクセスします。
2. 「アカウント」セクションから「作成」をクリックします。
3. 画面の指示に従い、必要な情報を入力します。
    - **業種・運用目的:** 「その他」を選択します。
    - **主な使い方:** 「メッセージ配信用」を選択します。
    - **ビジネスマネージャーの組織との接続方法:** 「ビジネスマネージャーの組織を選択」を選び、手順2で作成したプロバイダーを選択します。
4. 全ての情報を入力後、チャンネルを作成します。

## 4. アカウントの設定

1. **[LINE Official Account Manager](https://manager.line.biz/)** にて、作成したアカウントを選択し、「設定」画面に移動します。
2. **Messaging APIの設定:**
    - 「Messaging API」タブを選択します。
    - 「Messaging APIを利用する」をクリックします。
    - 利用するプロバイダーを選択します。
3. **アカウント設定:**
    - 「アカウント設定」タブに移動します。
    - **トークへの参加:** 「グループ・複数人トークへの参加を許可する」に変更します。
4. **応答設定:**
    - 「応答設定」タブに移動します。
    - **あいさつメッセージ:** OFFに設定します。
    - **応答メッセージ:** OFFに設定します。

## 5. Messaging API の有効化と設定

1. **[Line Developer Console](https://developers.line.biz/console/)** にアクセスし、対象のプロバイダーを選択します。
2. 作成したチャンネルがプロバイダーに登録されていることを確認します。このとき、「チャネル基本設定」タブに表示されている「**チャネルシークレット**」を控えておきます。
    - > **Note:** この値は、GCPのSecret Managerに `LINE_CHANNEL_SECRET` という名前で登録する必要があります。
3. **Webhook設定:**
    - 「Messaging API」タブを開きます。
    - **Webhook URL:** アプリケーションのWebhookエンドポイントURLを入力します。
    - 「検証」ボタンをクリックし、ステータスコード200が返却されることを確認します。
    - 「Webhookの利用」を有効（ON）にします。
4. **チャンネルアクセストークン:**
    - 「チャンネルアクセストークン」のセクションに移動します。
    - 「発行」ボタンをクリックして、新しいアクセストークンを発行します。
    - > **Note:** 発行されたトークンは、GCPのSecret Managerに `LINE_CHANNEL_ACCESS_TOKEN` という名前で登録する必要があります。
